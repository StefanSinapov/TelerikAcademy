03. Trees and Traversals
link from video
- Binary Trees - дървета който имат най-много две деца
- Binary Serch Trees - имаме подредба, и слагаме елементите в ляво или дясно спрямо строго дефинирано условие
- Balanced Binary Search Trees - дърво с подредба, но балансирано така че да има приблизително равен брой стъбла в ляво и дясно

Depth-first Search (DFS)
use Stack or recursion
DFS(node)
{
     for each child c of node
         DFS(c);
     print the current node;
}
pre-order: root, left, right
in-order: left, root, right
post-order: left, right, root

Breadth-First Search (BFS)
- използва се за намиране на най-краткия път
BFS(node)
{
       queue <- node
       while queue not empty
              v <- queue
         print v
         for each child c of v
                queue <- c
}

Balanced Search Trees
- AVL trees - ideally balanced, very complex (good search, a bit slow add)
- Red-black trees - roughly balanced, more simple (slow search, fast add)
- AA-Trees - simple to implement

04. Dictionaries, Hash Tables and Sets
ADT - Abstract Data Type
Hash Table
- an array that holds a set of (key, value) pairs
- чрез тях се имплементира речник
- hashing - това е процеса на свързване на ключ с някаква позиция
Hash Function
     - H(k) - хеш функция, която според нашия key какъвто и да е той тя ни връща int
     - h(k) = p and 0 ≤ p < m
Perfect Hashing function (PHF) - in most cases impossible
     - Chaining - правим масив от масиви или най-добре масив от LinkedLists

Dictionary<TKey, TValue>
- скорост: O(1)
Relies on : 
- Object.Equals() - for comparing the keys
- Object.GetHashCode() - calculating the hash code from keys
Operations:
.Add(TKey, TValue)
.Remove(Tkey)
this[] - get/add/replace element by key
.Clear() - removes all
.Count
.Keys - return collection of keys
.Values - return collection of the values
.ContainsKey(TKey) 
.ContainsValue(TValue)
.TryGetValue(Tkey, out TValue) - of key is found return is in the TValue otherwise returns false
DIctionary<string, List<int>>

 SortedDictionary<TKey, TValue> (Balanced Tree Dictionaries)
- Пази елементите сортирани по ключ
- скороста на операциите е O(log(n))
Relies on:
IComparable<T> for ordering the keys
- когато използваме наши класове за ключ, трябва да наследяват и да имплементират IComparable

Set ADT: 
- множества от стойности които не се повтарят
Operations:

Delete(element)

Bag ADT:

HashSet<T>

07. Sorting and Searching Algorithms
Sorting
Selection Sort - O(N*N)
Buble Sort - O(N*N)
Insertion Sort - O(N*N) - бърз за до 10 елемента
Quick sort
- Avg:  N*log(N) 
- Worst (N*N)
- много добър за повече елементи
- Stable - зависи
- Method - partitioning
Merge sort:
- Avg: N*log(N)
- Memory - worst case n
- Stable: Yes
Heap sort:
- N*log(N) 
- Stable: No
- Method: Seleciton
Counting sort:
- когато знаем само какви елементи ще имаме
- Avg: n + r      (r - range of numbers to be sorted)
- Memory: n + r
- Stable: Yes
- Method: non comparison
Bucket sort:
k - number of buckets
- Avg: n + k
- Worst: n*n + k
Searching:
Linear Search
- O(n)
Binary Search 
- търсене върху сортирана структура от данни
- O(log(n)) 
Interpolation Search
- Avg: log(log(n))
- worst: O(n)
Shuffling
- procedure used to randomize the order of items in collection

Fisher-Yetes algorithm - best
